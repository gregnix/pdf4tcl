Changes in v0.9.4.17 (2026-03-19, fork gregnix):
  • String encryption (ISO 32000 Â§7.6.5): EncryptStringsInBody encrypts all PDF literal strings in object dictionaries (/T, /DA, /V, /TU, /CA). Fixes AcroForm field corruption in encrypted documents.
  • SHA-384/512 pure-Tcl (::pdf4tcl::sha2pure, NIST FIPS 180-4). AES-256 no longer requires any external program. SHA backend priority: tcl-sha (C extension) > openssl (if in PATH) > pure-Tcl fallback.
  • Combobox and listbox generate own appearance streams; NeedAppearances is never set. All field types coexist correctly with signature fields.
  • New addForm options: -required, -label.
Changes in v0.9.4.16 (2026-03-17/18, fork gregnix):
  • Windows: twapi-Fix (twapi::sha384 returns SHA-1 not SHA-384 -- removed); demo files: non-ASCII replaced by \uXXXX for Windows cp1252 compatibility.
  • AES-256 encryption (V=5, R=6, PDF 2.0) via new option -encversion 5. Algorithm 2.B compatible with qpdf (password||K||ukey, sum(E[0:16])%3, K keeps full SHA-384/512 length in loop). SHA-384/512 via openssl stdin pipe (pure-Tcl fallback added in 0.9.4.17).
  • Bugfix: Algorithm 2.B K-truncation: K now keeps full 48/64 byte length after SHA-384/512 within the iteration loop.
  • pdflab2.tcl: AES-256 support (showEncDict, showEncryptVerify, H4/KRITISCH-2). New verify_enc3.py for AES-256 verification.
  • tests: new-0.9.4.16.test with 29 tests for AES-256 and string encryption; graceful skip when openssl or Tk unavailable.
  • Documentation: all doc/de/ and doc/en/ files updated to 0.9.4.16; AES-128 reference unified to PDF 1.6+; tests/README.md added.
Changes in v0.9.4.15 (2026-03-16, fork gregnix):
  • OTF/CFF font support in loadBaseTrueTypeFont: OpenType fonts with CFF outlines (.otf files, OTTO magic number) are now accepted. Previously raised TTF: postscript outlines are not supported. Font binary is embedded as-is; only metadata tables are read. PDF embedding uses /CIDFontType0 + /FontFile3 with /Subtype /OpenType instead of the TTF path (CIDFontType2 / FontFile2). New internal flag BFA($name,isCFF). 25 new tests in tests/new-0.9.4.15.test.
Changes in v0.9.4.14 (2026-03-15, fork gregnix):
  • New method addEmbeddedFile: embeds a file silently in the PDF Catalog /Names /EmbeddedFiles NameTree (ISO 32000 SS7.11.4). No visible page annotation is created. Options: -contents (raw data), -mimetype, -description, -afrelationship (Alternative | Data | Source | Supplement | Unspecified). Raises an error when -pdfa 1b is set (ISO 19005-1 SS6.1.7 forbids embedded files in PDF/A-1). Intended use: ZUGFeRD / Factur-X electronic invoices and other document-level attachments.
Changes in v0.9.4.13 (2026-03-15, fork gregnix):
  • New method setBlendMode: sets PDF blend mode (Normal, Multiply, Screen, Overlay, Darken, Lighten, ColorDodge, ColorBurn, HardLight, SoftLight, Difference, Exclusion, Hue, Saturation, Color, Luminosity) via ExtGState /BM. Requires PDF 1.4 (raised automatically).
  • New method getBlendMode: returns currently active blend mode.
  • New method linearGradient: axial gradient fill (ShadingType 2 + FunctionType 2). Options: -extend.
  • New method radialGradient: radial gradient fill between two circles (ShadingType 3 + FunctionType 2). Options: -extend.
  • New options -tooltip (/TU) and -tabindex (/TI) in addForm for PDF/UA accessibility and AcroForm tab order.
  • endPage now writes /Tabs /R in the page dict when annotations are present.
  • Bugfix: method DrawRect header was accidentally dropped; rectangle calls would have failed with "unknown method DrawRect".
  • Bugfix: getBlendMode always returned "Normal" (missing assignment in setBlendMode).

Changes in v0.9.1 (2017-03-02):
  • Requires Tcl 8.6. Does not require Snit.
  • Added -compress option to addRawImage and putRawImage.
  • Added -angle and -anchor options to putImage and putRawImage.
  • Fixed bug where an unnecessary startPage was called from addImage and addRawImage.
  • Added pdf4tcl::getFonts call.
  • Added -dryrun option to drawTextBox method.
Incompatible changes in v0.9:
  • Pdf4tcl objects are now TclOO objects. Some of the ways to create an object is therefore different and will not work. The documented way, using pdf4tcl::new, should work as before.
  • Default value of document option -compress is now true.
  • Removed deprecated functionality:
    • Method cleanup removed (use destroy)
    • Method qCurve removed (use curve)
    • Method drawText removed (use text and newLine)
    • Method addJpeg removed (use addImage)
    • Non-option arguments to startPage removed (use options)
Changes in v0.8.4 (2016-08-15):
  • Note: 0.8.4 will be the last version to support Tcl 8.4/8.5, and the last to require Snit. Next will require TclOO and other 8.6 features.
  • Adapted to tkpath items, using itempdf support in tkpath.
  • Support for characters outside BMP, if run in a Tcl-version supporting it.
  • Added addForm method, for creating interactive forms.
  • Added catPdf command.
  • setLineStyle, setLineWidth and setLineDash now interprets their inputs with units, like all distances
  • Added document flag -cmyk and started on CMYK support.
  • Added startXObject method, for creating Form XObjects.
  • Added embedFile and attachFile methods.
Changes in v0.8 (2012-06-12):

  • Added setLineWidth and setLineDash methods.
  • Fixed bug "that makes non-latin Italic TTF-fonts unusable".
Incompatible changes in v0.8:

  • Deprecated: qCurve.
  • Cleaned up to name ascend/descent font metrics consistently.
Changes in v0.7 (2010-01-30):

Changes in v0.4 (2008-03-04):
  • New method canvas for dumping canvas contents to PDF. Demo

  • New methods putRawImage, addRawImage.
  • New methods moveTextPosition, setLineSpacing, getLineSpacing.

Changes in v0.3 (2008-01-19):
  • Added -file option to continuously write to file instead of accumulating in memory.
  • More font metric introspection.
  • More general handling of page properties like size and margins.
  • Draw text with background fill.
  • All user coordinates support e.g. "5mm" as input.
  • Redesigned to use Snit to get clearer code. This caused a noticable slowdown but since Snit will be more than twice as fast with 8.5 this will probably be regained.
