Changes in v0.9.4.17 (2026-03-19, fork gregnix):
  • Tcl 9 compatibility: pdf4tcl now runs under both Tcl 8.6 and Tcl 9.0
  • encrypt.tcl: SHA backend auto-selects correct path under Tcl 9 (tcl-sha 9.0 build)
  • encrypt.tcl: SHA output via hex + binary decode hex -- avoids corrupted binary data under Tcl 9
  • encrypt.tcl: new _AesCbc/_AesEcb wrappers use binary format a* for guaranteed bytearrays
  • helpers.tcl: CleanText handles unencodable characters char-by-char (Tcl 9 throws instead of replacing)
  • main.tcl: write method adds -encoding iso8859-1 to fix EILSEQ under Tcl 9
  • main.tcl: _ValidatePdfDate checks Unix timestamp range to handle Tcl 9 big-integer behaviour
  • Makefile: TCLSH ?= allows override via environment (tclsh8.6 or tclsh9.0)
  • tests: -encoding binary replaced by -translation binary (Tcl 9)
  • tests: example PDFs auto-regenerated when Tcl version changes
  • tools: bump.tcl rewritten -- lean, reads from next.tcl, no blocking consistency check
  • tools: tcl9env.sh added -- sets Tcl 9 environment for make test/example
  • tcl-sha 2.1.1 ported to Tcl 9 (Tcl_Size shim, string totitle in pkgIndex.tcl)
